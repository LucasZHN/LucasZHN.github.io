/etc/rear/local.conf Configuration

find BACKUP=NETFS, change or append:

  • BACKUP_URL=cifs://172.16.4.27/dsa14/ -> BACKUP_URL=file:///mnt/usb
  • BACKUP_OPTIONS=”cred=/etc/rear/cifs” -> BACKUP_OPTIONS=

Preparation for Upgrade

Create a root SSH key

  • Copy the SSH (private) keys to the maintenance PC for later use.

prodis-playbooks*.rpms

Installation

Login to the passive Server, check if the system is in good health

Switch the cluster into maintenance mode and the passive server into standby.

Create a backup of file /etc/rear/local.conf

Start the PRODIS.Server Installer

CHECK->UPLOAD->INSTALL

In case of any problems, ansible log is at /var/log/ansible.log.

Install RR2021-1-HF4 on PRODIS.Server and PRODIS.StatServer

Quick Test

  1. prodis.service -> no red or no dead services or failed.
  1. Check database -> List all vehicles in the database.
  1. Check communication to tester -> no errors should appear.
  1. Check queue status ->queues should be empty.
  1. Check system monitoring -> OK or TIMEOUT
  1. Cluster status
